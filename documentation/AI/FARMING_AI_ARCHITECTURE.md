## 1. Vision & Principles

### What FarmBrain Is
FarmBrain is a **fully proprietary, PostgreSQL-driven agricultural intelligence engine** built and owned by 4Directions. It is not a wrapper around any third-party LLM (not OpenAI, not Claude, not Gemini). Every model, every weight, every byte of training data is controlled entirely by 4Directions.

### Core Principles

| Principle | Meaning |
|-----------|---------|
| **Own the data** | All training data is collected, licensed, or generated by 4Directions |
| **Own the model** | All models are trained in-house — no API dependency |
| **Farming only** | The model vocabulary, knowledge, and reasoning are 100% agriculture-focused |
| **PostgreSQL-first** | The database is not just storage — it IS the intelligence backbone |
| **On-device capable** | Critical models run on the farmer's phone without internet |
| **South Africa first** | All data, crop lists, weather zones, regulations, and pricing are SA-primary |
| **Language-aware** | Supports English, Zulu, Sotho, Afrikaans farm terminology |

### What FarmBrain Delivers
```
Farmer asks: "My maize leaves have yellow stripes and the plant is small"
FarmBrain:
  → Retrieves from vector DB: disease symptoms matching yellow stripes + stunting
  → Cross-references: current weather (warm, humid = MSV risk)
  → Cross-references: farm location (Limpopo, leafhopper pressure)
  → Cross-references: crop stage (V4, vulnerable to MSV)
  → Generates: "This matches Maize Streak Virus (85% confidence).
               Leafhoppers active in your area. Immediate action required.
               Registered insecticide: Actara 25 WG. See treatment protocol."
```
All from data the farmer themselves contributed — and SA agricultural sources.

## 3. PostgreSQL as the AI Data Brain

PostgreSQL is not just the application database — it is the **living intelligence layer** that feeds, trains, and serves FarmBrain.

### 3.1 Core PostgreSQL Extensions Required

### 3.3 Intelligence Views — Real-Time AI Context

```sql
-- View: Farm context snapshot for the advisor engine
CREATE MATERIALIZED VIEW farmbrain.farm_ai_context AS
SELECT
    f.id                    AS farm_id,
    f.province,
    f.district,
    f.soil_type,
    f.hectares,
    f.irrigation_type,
    f.annual_rainfall_mm,
    -- Latest weather (last 24h)
    w.temp_c                AS current_temp_c,
    w.humidity_pct          AS current_humidity,
    w.rainfall_mm           AS rainfall_24h,
    w.spray_window,
    w.frost_risk,
    -- 7-day rainfall sum
    (
        SELECT COALESCE(SUM(rainfall_mm), 0)
        FROM farmbrain.weather_readings wr
        WHERE wr.farm_id = f.id
          AND wr.time >= now() - INTERVAL '7 days'
    )                       AS rainfall_7d_mm,
    -- Latest soil pH
    (
        SELECT ph FROM farmbrain.soil_analyses sa
        WHERE sa.farm_id = f.id
        ORDER BY sample_date DESC LIMIT 1
    )                       AS latest_soil_ph,
    -- Active pest observations count
    (
        SELECT COUNT(*) FROM farmbrain.farm_observations fo
        WHERE fo.farm_id = f.id
          AND fo.obs_type IN ('disease', 'pest')
          AND fo.obs_date >= now() - INTERVAL '30 days'
    )                       AS active_pest_obs_count,
    -- Average yield last 3 seasons
    (
        SELECT ROUND(AVG(yield_t_ha)::NUMERIC, 2)
        FROM farmbrain.yield_records yr
        WHERE yr.farm_id = f.id
          AND yr.season_year >= EXTRACT(YEAR FROM now()) - 3
    )                       AS avg_yield_3yr_t_ha,
    -- Latest market price for primary crop
    (
        SELECT price_zar_t FROM farmbrain.market_prices mp
        ORDER BY time DESC LIMIT 1
    )                       AS latest_market_price_zar_t
FROM farmbrain.farms f
LEFT JOIN LATERAL (
    SELECT * FROM farmbrain.weather_readings wr
    WHERE wr.farm_id = f.id
    ORDER BY time DESC LIMIT 1
) w ON TRUE;

-- Refresh every 30 minutes via pg_cron
SELECT cron.schedule('refresh-farm-ai-context', '*/30 * * * *',
    'REFRESH MATERIALIZED VIEW CONCURRENTLY farmbrain.farm_ai_context');

-- ──────────────────────────────────────────────────────────────────────
-- Semantic disease search function (pgvector)
-- Used by the advisor to retrieve relevant disease knowledge
-- ──────────────────────────────────────────────────────────────────────
CREATE OR REPLACE FUNCTION farmbrain.search_diseases(
    query_embedding  vector(768),
    crop_filter      TEXT DEFAULT NULL,
    match_threshold  FLOAT DEFAULT 0.75,
    match_count      INT DEFAULT 5
)
RETURNS TABLE (
    disease_id      TEXT,
    disease_name    TEXT,
    similarity      FLOAT,
    visual_symptoms TEXT,
    category        TEXT,
    severity_level  TEXT
) AS $$
    SELECT
        d.id,
        d.disease_name,
        1 - (d.embedding <=> query_embedding) AS similarity,
        d.visual_symptoms,
        d.category,
        d.severity_level
    FROM farmbrain.disease_knowledge d
    WHERE
        (crop_filter IS NULL OR crop_filter = ANY(d.crop_ids))
        AND 1 - (d.embedding <=> query_embedding) > match_threshold
    ORDER BY d.embedding <=> query_embedding
    LIMIT match_count;
$$ LANGUAGE sql STABLE;

-- ──────────────────────────────────────────────────────────────────────
-- RAG knowledge retrieval function
-- ──────────────────────────────────────────────────────────────────────
CREATE OR REPLACE FUNCTION farmbrain.retrieve_knowledge(
    query_embedding  vector(768),
    topic_filter     TEXT[] DEFAULT NULL,
    crop_filter      TEXT[] DEFAULT NULL,
    lang_filter      TEXT DEFAULT 'en',
    match_count      INT DEFAULT 8
)
RETURNS TABLE (
    chunk_id   UUID,
    source_type TEXT,
    title      TEXT,
    chunk_text TEXT,
    similarity FLOAT,
    topic_tags TEXT[]
) AS $$
    SELECT
        k.id,
        k.source_type,
        k.title,
        k.chunk_text,
        1 - (k.embedding <=> query_embedding) AS similarity,
        k.topic_tags
    FROM farmbrain.knowledge_chunks k
    WHERE
        k.language = lang_filter
        AND (topic_filter IS NULL OR k.topic_tags && topic_filter)
        AND (crop_filter IS NULL OR k.crop_ids && crop_filter)
        AND k.quality_score > 0.6
    ORDER BY k.embedding <=> query_embedding
    LIMIT match_count;
$$ LANGUAGE sql STABLE;
```

## 6. FarmBrain Model Architecture

### 6.1 Model 1 — FarmVision (Disease & Crop Detection)

```
Architecture: EfficientNetV2-S + Custom Agricultural Head

Input:  224 × 224 × 3 RGB leaf/plant image
Output: [disease_class_probabilities] + [crop_type] + [severity_score]

┌─────────────────────────────────────────────────────────────────┐
│  EfficientNetV2-S Backbone (pretrained ImageNet, fine-tuned)    │
│  Parameters: ~21M                                                │
│  Output: 1280-dim feature vector                                │
├─────────────────────────────────────────────────────────────────┤
│  Global Average Pooling 2D                                      │
├─────────────────────────────────────────────────────────────────┤
│  Dense(512) + BatchNorm + ReLU + Dropout(0.4)                   │
├───────────────────────────────┬─────────────────────────────────┤
│  HEAD 1: Disease Classes      │  HEAD 2: Severity Score         │
│  Dense(num_diseases)          │  Dense(1) + Sigmoid             │
│  + Softmax                    │  Output: 0.0 – 1.0              │
│  Output: ~60 SA diseases      │                                 │
└───────────────────────────────┴─────────────────────────────────┘

Training Config:
  - Phase 1: Freeze backbone, train heads — 20 epochs, lr=1e-3
  - Phase 2: Unfreeze top 30 layers, fine-tune — 30 epochs, lr=1e-5
  - Loss: Categorical Crossentropy + MSE (multi-head)
  - Augmentation: RandomFlip, RandomRotation, ColorJitter, CutMix
  - Target accuracy: > 90% top-1, > 98% top-3

On-device:
  - Quantized to INT8 via TFLite
  - Model size: ~6 MB
  - Inference time: < 200ms on mid-range Android
```

### 6.2 Model 2 — FarmYield (Yield Prediction)

```
Architecture: Gradient Boosted Trees + LSTM hybrid

Input Features (45 features):
  Soil:     pH, N, P, K, Ca, Mg, Zn, OC%, CEC, soil_type (encoded)
  Weather:  GDD_season, rainfall_total, frost_days, drought_index,
            solar_rad_avg, temp_max_avg, temp_min_avg
  Farm:     irrigation_type, hectares, altitude, province (encoded)
  Crop:     variety (encoded), days_since_planting, growth_stage_encoded
  Inputs:   fertilizer_n_kg_ha, fertilizer_p_kg_ha, fertilizer_k_kg_ha,
            pesticide_applications, irrigation_mm

Historical:
  LSTM(64 units) over last 5 seasons' yield + weather sequences

Output:
  - Predicted yield (t/ha) with confidence interval
  - Feature importance (which factors drive yield most on this farm)

Target RMSE: < 0.4 t/ha for maize
```

### 6.3 Model 3 — FarmBrain LLM (Agricultural Advisor)

```
Architecture: Fine-tuned T5-Base (250M parameters)
Strategy: Domain-adaptive pre-training → instruction fine-tuning

Stage 1: Domain-Adaptive Pre-training (DAPT)
  Corpus: All agricultural documents from Tier 4 sources
  Objective: Masked language modeling on farming text
  Duration: ~2 weeks on 4× A100 GPUs
  Output: T5-FarmBase (agriculture-adapted weights)

Stage 2: Instruction Fine-tuning
  Dataset: 50 000+ farming Q&A pairs constructed from:
    - ARC extension officer FAQ databases
    - DALRRD farmer helpline transcripts
    - Grain SA advisor sessions
    - 4Directions advisor session logs (anonymised)
  Format:
    Input:  "Context: {farm_context}\nQuestion: {farmer_question}"
    Output: "{structured_recommendation}"
  Duration: ~3 days on 4× A100 GPUs

Stage 3: RLHF (Reinforcement Learning from Human Feedback)
  Reward signal: Farmer satisfaction ratings (1–5 stars)
  Feedback collected via: App advisor session rating
  Frequency: Quarterly fine-tuning cycle

On-device option:
  - Distill T5-FarmBase to T5-Small (60M parameters)
  - Quantize to INT4 → ~30 MB TFLite model
  - Works offline for common queries
```

### 6.4 Model 4 — FarmEmbed (Farming Sentence Embeddings)

```
Architecture: Fine-tuned sentence-transformers/all-mpnet-base-v2

Purpose:
  - Convert farmer descriptions to vectors for semantic search
  - Power the pgvector RAG system
  - Disease symptom matching

Training:
  - Positive pairs: (symptom description, disease name)
  - Negative pairs: (symptom description, wrong disease)
  - Loss: MultipleNegativesRankingLoss
  - Dataset: 25 000 expert-curated symptom–disease pairs

Output dimension: 768
Stored in: farmbrain.disease_knowledge.embedding
           farmbrain.knowledge_chunks.embedding
```

### 6.5 Model 5 — FarmSpray (Spray Decision Engine)

```
Architecture: XGBoost binary/multi-class classifier

Input (14 features):
  wind_kmh, humidity_pct, temp_c, rain_forecast_24h_mm,
  rain_forecast_48h_mm, uv_index, time_of_day_h,
  crop_growth_stage_encoded, leaf_wetness_pct,
  soil_moisture_pct, product_type_encoded,
  application_rate_l_ha, nozzle_type_encoded,
  wind_direction_encoded

Output: SprayWindow (0=unsuitable, 1=marginal, 2=suitable)
        + optimal_time_window (e.g., "06:00–09:00")

Training data:
  - Historical SAWS weather + spray records from app
  - CROPLIFE SA spray guidelines
  - Insecticide resistance management thresholds

Target accuracy: > 95%
```

```python
    conn = psycopg2.connect(settings.POSTGRES_URL)
    df = pd.read_sql("""
        SELECT
            fo.image_paths[1]   AS image_path,
            fo.disease_id       AS label,
            fo.growth_stage,
            fo.farm_id,
            f.province,
            dk.category,
            dk.severity_level
        FROM farmbrain.farm_observations fo
        JOIN farmbrain.farms f ON fo.farm_id = f.id
        JOIN farmbrain.disease_knowledge dk ON fo.disease_id = dk.id
        WHERE fo.expert_verified = TRUE
          AND fo.image_paths IS NOT NULL
          AND array_length(fo.image_paths, 1) > 0
        ORDER BY fo.created_at DESC
    """, conn)
    return df

def build_model(num_classes: int) -> tf.keras.Model:
    base = tf.keras.applications.EfficientNetV2S(
        weights='imagenet', include_top=False,
        input_shape=(224, 224, 3)
    )
    x = tf.keras.layers.GlobalAveragePooling2D()(base.output)
    x = tf.keras.layers.Dense(512, activation='relu')(x)
    x = tf.keras.layers.BatchNormalization()(x)
    x = tf.keras.layers.Dropout(0.4)(x)
    output = tf.keras.layers.Dense(num_classes, activation='softmax')(x)
    return tf.keras.Model(base.input, output)

def train():
    with mlflow.start_run(run_name="disease_classifier_v3"):
        df = load_training_data_from_postgres()

        # Log dataset statistics

            "total_samples": len(df),
            "num_classes": df['label'].nunique(),
            "provinces": df['province'].unique().tolist(),

        for layer in model.layers[:-8]:

        model.compile(optimizer=tf.keras.optimizers.Adam(1e-3),

                      metrics=['accuracy'])
        model.fit(train_ds, validation_data=val_ds, epochs=20,
```

## 11. Retrieval-Augmented Generation (RAG)

The RAG system makes the FarmBrain LLM accurate on up-to-date SA-specific farming knowledge without requiring it to memorise everything during training.

```python
# farmbrain_api/rag/retriever.py

class FarmBrainRAG:
    """
    PostgreSQL + pgvector powered RAG system.
    Retrieves the most relevant agricultural knowledge chunks
    to ground every LLM response.
    """

    def __init__(self, db_conn, embedder: FarmBrainEmbedder):
        self.db = db_conn
        self.embedder = embedder

    def retrieve(
        self,
        query: str,
        farm_context: dict,
        topic: str,
        top_k: int = 8
    ) -> list[KnowledgeChunk]:
        # 1. Embed the query
        query_embedding = self.embedder.encode(query)

        # 2. Build filters from farm context
        crop_filter = farm_context.get('active_crop_ids', None)
        topic_filter = [topic] if topic else None

        # 3. Semantic search via pgvector
        chunks = self.db.execute("""
            SELECT * FROM farmbrain.retrieve_knowledge(
                query_embedding => %(emb)s::vector,
                topic_filter    => %(topics)s,
                crop_filter     => %(crops)s,
                lang_filter     => %(lang)s,
                match_count     => %(k)s
            )
        """, {
            'emb': query_embedding.tolist(),
            'topics': topic_filter,
            'crops': crop_filter,
            'lang': farm_context.get('language', 'en'),
            'k': top_k
        }).fetchall()

        # 4. Hybrid re-ranking: semantic score + topic relevance + recency
        return self._rerank(chunks, query, farm_context)

    def build_prompt(
        self,
        query: str,
        chunks: list[KnowledgeChunk],
        farm_context: dict
    ) -> str:
        knowledge_text = "\n\n---\n\n".join([
            f"[Source: {c.source_type} | Topic: {', '.join(c.topic_tags)}]\n{c.chunk_text}"
            for c in chunks
        ])

        return f"""You are FarmBrain, an agricultural AI advisor for South African farmers.
You only provide advice based on the knowledge provided below.
You speak directly to the farmer. Be practical, specific, and concise.

FARM CONTEXT:
Province: {farm_context['province']}
Crop: {farm_context.get('crop_type', 'Not specified')}
Current temperature: {farm_context.get('current_temp_c', 'Unknown')}°C
Rainfall last 7 days: {farm_context.get('rainfall_7d_mm', 0)}mm
Soil pH: {farm_context.get('latest_soil_ph', 'Unknown')}
Spray window: {farm_context.get('spray_window', 'Unknown')}

FARMING KNOWLEDGE BASE:
{knowledge_text}

FARMER'S QUESTION:
{query}

RESPONSE (in clear, practical farming language):"""
```

### 18.1 API Framework — Why FastAPI, Not Django or Flask

| Criteria | FastAPI | Django REST | Flask |
|----------|---------|-------------|-------|
| **Performance** | ★★★★★ Async ASGI, fastest Python | ★★★ Sync by default | ★★★ Sync by default |
| **Auto OpenAPI docs** | ★★★★★ Built-in, live Swagger | ★★★ drf-spectacular add-on | ★★ Flask-RESTX add-on |
| **Type safety** | ★★★★★ Pydantic v2 models | ★★★ Serializers (manual) | ★★ No enforced typing |
| **ML integration** | ★★★★★ Native NumPy/PyTorch | ★★★ Possible but heavy ORM | ★★★ Lightweight, flexible |
| **Async support** | ★★★★★ Native `async def` | ★★ ASGI mode (Django 4.1+) | ★★ Flask async (3.0+) |
| **Development speed** | ★★★★ Minimal boilerplate | ★★★★★ Admin, auth built-in | ★★★★ Very simple |
| **Production maturity** | ★★★★ Widely used in ML | ★★★★★ Battle-tested | ★★★★ Battle-tested |

**Decision: FastAPI** — The AI API handles image tensors, embedding arrays, and LLM token streams. Async I/O prevents blocking while GPU inference runs. Pydantic v2 validates all request/response shapes automatically. Django's ORM is unnecessary overhead when PostgreSQL logic is handled via raw SQL + pgvector functions.

```python
# Example: The async advantage — non-blocking LLM inference
@app.post("/v1/advisor/query")
async def advisor_query(request: AdvisorRequest, db=Depends(get_db)):
    # These run concurrently, not sequentially:
    embedding_task = asyncio.create_task(embedder.aencode(request.question))
    farm_ctx_task  = asyncio.create_task(db.afetch_one(FARM_CTX_SQL, request.farm_id))

    query_embedding, farm_ctx = await asyncio.gather(embedding_task, farm_ctx_task)

    # Non-blocking LLM call via vLLM async API
    response = await vllm_client.generate(
        build_prompt(request.question, farm_ctx, query_embedding)
    )
    return AdvisorResponse(**response)
```

### 18.2 Training Framework — PyTorch Primary, TensorFlow Export Only

**Why PyTorch for all training:**
- Native Python control flow — debugging a training loop in PyTorch is just Python debugging
- `torch.compile()` (PyTorch 2.0+) matches TensorFlow XLA speed
- Hugging Face Transformers, PEFT, and vLLM are PyTorch-native
- More research papers release PyTorch code first — adapting SA-specific architectures is faster

**Why TensorFlow/TFLite at all:**
- `tflite_flutter` on Flutter only supports `.tflite` model format
- TFLite INT8 quantization produces the smallest mobile models (6 MB for FarmVision)
- The conversion is one command: `torch → ONNX → TFLite` via `ai_edge_torch`

### 18.3 LLM Fine-Tuning — PEFT + LoRA Strategy

Training T5-Base (250M parameters) fully would require 4× A100s for weeks. LoRA (Low-Rank Adaptation) inserts small trainable matrices into the frozen model — only ~0.1% of parameters are trained.

```
Full fine-tune:  250M parameters × float32 = ~1 GB GPU memory per parameter copy
                 Typical: 4× A100 (320 GB total) needed

LoRA fine-tune:  ~250K trainable parameters (rank=16, ~64 layers)
                 Typical: 1× A100 (80 GB) sufficient
                 Cost: ~$50 total vs ~$2000+ for full fine-tune
```

```python
# Fine-tuning FarmBrain LLM with PEFT + LoRA
from peft import LoraConfig, get_peft_model, TaskType
from transformers import T5ForConditionalGeneration, T5Tokenizer

# Load base T5-Base
model = T5ForConditionalGeneration.from_pretrained("t5-base")
tokenizer = T5Tokenizer.from_pretrained("t5-base")

# Apply LoRA adapters — only these layers train
lora_config = LoraConfig(
    task_type=TaskType.SEQ_2_SEQ_LM,
    r=16,                        # LoRA rank — higher = more capacity
    lora_alpha=32,               # Scaling factor
    target_modules=["q", "v"],   # Attention query and value matrices
    lora_dropout=0.05,
    bias="none"
)
model = get_peft_model(model, lora_config)
model.print_trainable_parameters()
# → trainable params: 294,912 || all params: 248,151,552 || trainable%: 0.12

# Training dataset — SA farming Q&A pairs
dataset = load_farming_qa_dataset()  # 50 000 pairs from PostgreSQL

# Fine-tune — runs on a single A100 in ~6 hours
trainer = Seq2SeqTrainer(
    model=model,
    args=Seq2SeqTrainingArguments(
        output_dir="./farmbrain-lora-v1",
        num_train_epochs=5,
        per_device_train_batch_size=16,
        gradient_accumulation_steps=4,
        warmup_steps=500,
        learning_rate=3e-4,
        fp16=True,               # Mixed precision — halves memory use
        predict_with_generate=True,
        generation_max_length=256,
    ),
    train_dataset=dataset['train'],
    eval_dataset=dataset['validation'],
)
trainer.train()

# Save only the LoRA adapter weights (~12 MB, not 1 GB)
model.save_pretrained("./farmbrain-lora-v1-adapter")
```

### 18.4 LLM Serving — vLLM, Not TorchServe or Ollama

| Serving Solution | Throughput | Batch Strategy | Best For |
|-----------------|-----------|---------------|---------|
| **vLLM** | ★★★★★ 10–20× HuggingFace | Continuous batching + PagedAttention | High-volume production API |
| **TorchServe** | ★★★ Good | Static batching | Vision/tabular models |
| **Ollama** | ★★ Local use | Single request | Local dev, demos |

| **TGI (Text Generation Inference)** | ★★★★ Near vLLM | Continuous batching | Alternative to vLLM |

**Decision: vLLM** for the FarmBrain LLM, TorchServe for FarmVision and FarmYield.

```python
# Airflow DAG — daily training data preparation
from airflow.decorators import dag, task
from datetime import datetime, timedelta

@dag(
    schedule_interval='0 01 * * *',   # Every day at 01:00 SAST
    start_date=datetime(2025, 1, 1),
    catchup=False,
    tags=['farmbrain', 'training']
)
def farmbrain_daily_pipeline():

    @task()
    def check_new_verified_observations() -> int:
        """Count newly verified disease images since last run."""
        return db.execute("""
            SELECT COUNT(*) FROM farmbrain.farm_observations
            WHERE expert_verified = TRUE
              AND used_in_training = FALSE
              AND image_paths IS NOT NULL
        """).scalar()

    @task()
    def ingest_saws_weather():
        """Pull latest SAWS weather data into TimescaleDB."""
        client = SAWSClient(api_key=settings.SAWS_KEY)
        readings = client.get_all_stations_latest()
        bulk_insert_weather(readings)

    @task()
    def update_market_prices():
        """Scrape SAFEX spot prices and futures."""
        prices = SAFEXScraper().fetch_latest()
        upsert_market_prices(prices)

    @task.branch()
    def should_retrain(new_count: int) -> str:
        return 'trigger_disease_retraining' if new_count >= 500 else 'skip_retraining'

    @task()
    def trigger_disease_retraining():
        """Submit training job to GPU cluster."""
        submit_ray_job(
            entrypoint="python train/disease_classifier.py --version auto",
            runtime_env={'working_dir': './training'}
        )

    @task()
    def skip_retraining():
        pass  # Not enough new data yet

    # Wire the DAG
    count = check_new_verified_observations()
    ingest_saws_weather()
    update_market_prices()
    branch = should_retrain(count)
    branch >> [trigger_disease_retraining(), skip_retraining()]

farmbrain_daily_pipeline()
```

### 18.6 Model Monitoring — Evidently AI for Drift Detection

Model performance degrades silently in production as SA crop conditions, new pest strains, or seasonal changes shift the data distribution. Evidently AI monitors this automatically.

```python
# monitoring/drift_monitor.py
import evidently
from evidently.report import Report
from evidently.metric_preset import DataDriftPreset, ClassificationPreset

def run_weekly_drift_check():
    """
    Compare this week's disease detection inputs against training baseline.
    If drift detected → alert ML team → schedule early retraining.
    """
    # Reference dataset: training distribution
    reference = load_training_feature_distribution('disease_classifier_v3')

    # Current dataset: last 7 days of production predictions
    current = db.query("""
        SELECT
            fo.growth_stage,
            w.temp_c,
            w.humidity_pct,
            w.rainfall_mm,
            f.province,
            fo.disease_id AS predicted_label
        FROM farmbrain.farm_observations fo
        JOIN farmbrain.farms f ON fo.farm_id = f.id
        JOIN farmbrain.weather_readings w ON w.farm_id = fo.farm_id
          AND w.time BETWEEN fo.created_at - INTERVAL '1 hour'
                         AND fo.created_at + INTERVAL '1 hour'
        WHERE fo.created_at >= now() - INTERVAL '7 days'
    """)

    # Generate drift report
    report = Report(metrics=[
        DataDriftPreset(),
        ClassificationPreset(),
    ])
    report.run(reference_data=reference, current_data=current)

    # Save report to S3 and check for alerts
    report.save_html(f's3://farmbrain-reports/drift/{date.today()}.html')

    if report.as_dict()['metrics'][0]['result']['dataset_drift']:
        send_alert_to_ml_team(
            subject="⚠️ FarmVision: Data drift detected",
            body=f"Disease detection inputs drifted from training distribution. "
                 f"Review drift report: s3://farmbrain-reports/drift/{date.today()}.html"
        )
```
